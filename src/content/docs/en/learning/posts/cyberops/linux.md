## MODULE OVERVIEW

Linux is an open-source, modular operating system characterized by its reliability, low hardware requirements, and high degree of customizability. In a **Security Operations Center (SOC)**, Linux is the preferred platform because its open-source nature allows for the creation of tailor-built environments specifically for security analysis (e.g., **Security Onion**). The operating system is designed with inherent network connectivity, facilitating the development and deployment of network-based security applications.



The **Command Line Interface (CLI)**, or **shell**, serves as the primary command interpreter, providing a powerful mechanism for analysts to interact with the system both locally and remotely. A critical security feature of Linux is the granular control provided by the **root user** (superuser), who possesses absolute authority over the system, including low-level functions such as the network stack.

## CORE CONCEPTS & DEFINITIONS

* **Linux Distribution (Distro)**: A package containing the Linux kernel bundled with specific tools and software. Examples include Debian, Red Hat, Ubuntu, and specialized distros like **Kali Linux** (Penetration Testing) or **Security Onion** (Blue Teaming).
* **Shell**: The command-line environment (e.g., Bash, Zsh) that interprets user-inputted commands.
* **Processes**: Programs currently executing in the system, often running in the background as **daemons** (services).
* **Mounting**: The process of linking a physical partition on a block device (Hard Drive, USB) to a directory (mount point) to make it accessible to the OS.
* **Log Files**: Records generated by software, services, or the OS to track specific events for troubleshooting and security auditing.

### Linux Filesystem Permissions
Permissions are managed through a triad of entities: **User** (Owner), **Group**, and **Others**. These are often represented in octal notation.

| Octal | Binary | Permission Type | Description |
| :--- | :--- | :--- | :--- |
| 4 | 100 | Read (r) | Allows viewing file contents or listing directory contents. |
| 2 | 010 | Write (w) | Allows modifying file contents or adding/deleting files. |
| 1 | 001 | Execute (x) | Allows running a file or entering (`cd`) a directory. |
| 7 | 111 | Full Access | Combination of Read, Write, and Execute (4+2+1). |
| 0 | 000 | No Access | All permissions are revoked. |

## OPERATIONAL ANALYSIS

### Process and Server Identification
To identify active servers and services, analysts utilize the `ps` and `netstat` (or `ss`) commands. The `sudo ps -elf` command provides a detailed list of all running processes, including the **Process ID (PID)** and **Parent Process ID (PPID)**.

**Critical Port Mapping (Attack Surface):**
* **Port 21**: FTP (File Transfer Protocol - Insecure/Cleartext).
* **Port 22**: **SSH** (Secure Shell - Encrypted remote access).
* **Port 23**: **Telnet** (**Vulnerable** - Remote access in cleartext/unencrypted).
* **Port 25**: SMTP (Simple Mail Transfer Protocol).
* **Port 53**: **DNS** (Domain Name System - UDP/53).
* **Port 80**: HTTP (Web traffic - Unencrypted).
* **Port 443**: HTTPS (Web traffic - Encrypted via TLS/SSL).

### Log File Investigation
Log files are typically stored in the `/var/log` directory. Effective analysis requires understanding the five core components of a log entry:
1.  **Timestamp**: Critical for event correlation and timeline reconstruction.
2.  **Type**: Categorizes the event (e.g., error, info, warning).
3.  **PID**: Identifies the specific process instance that generated the log.
4.  **Client/Source**: Records the IP address or user initiating the transaction.
5.  **Description**: Detailed technical information regarding the event.

```bash
# Example: Monitoring system logs in real-time
sudo tail -f /var/log/messages
```

## CASE STUDIES & EXAM SPECIFICS

### Hardening and Vulnerability Assessment
* **Penetration Testing**: The process of attacking a network to find vulnerabilities. This is **offensive**, unlike an **IDS**, which is a **defensive** monitoring tool.
* **Device Hardening**: Includes disabling unnecessary services, enforcing strong password policies, and restricting physical access (e.g., disabling USB auto-detection).
* **Symbolic vs. Hard Links**: 
    * **Symbolic Links (`ln -s`)**: Point to a filename. If the original file is renamed, the link breaks.
    * **Hard Links (`ln`)**: Point to the **inode** (the actual data block on disk). They remain functional even if the original filename is changed.

### Command Reference for the Shell
* `man [command]`: Accesses the built-in documentation (manual pages).
* `pwd`: Displays the **Print Working Directory**.
* `ls -l`: Lists directory contents with long-format details (permissions, owners).
* `lsblk`: Displays block devices and their mount points.
* `cp / mv`: Copy / Move or Rename files and directories.
* `sudo`: Executes commands with superuser (root) privileges.
* `|` (Pipe): Combines two commands by sending the output of the first as input to the second (e.g., `ps -elf | grep telnet`).
